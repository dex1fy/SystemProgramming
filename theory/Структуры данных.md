## Структуры данных. Актуальность использования структур. Передача структуры как параметра.

**Структуры — это "пакет с данными".**

---

### **Зачем нужны?**
1. **Собрать связанное вместе**
   ```c
   // Вместо 3 отдельных переменных:
   int x = 10;
   int y = 20;
   int z = 30;
   
   // Одна структура:
   struct Point {
       int x, y, z;
   };
   struct Point p = {10, 20, 30};
   ```

2. **Упростить передачу в функции**
   - Вместо 10 параметров — 1 структура

3. **Вернуть несколько значений** из функции

---

### **Как передавать в функции?**

#### **1. По значению (копия) — редко**
```c
void func(struct Point p) {  // Делается ПОЛНАЯ копия
    p.x = 100;  // Меняем копию
}
// Оригинал не меняется
// Медленно для больших структур
```

#### **2. По указателю (чаще всего) ✓**
```c
void func(struct Point* p) {  // Передаём АДРЕС (4/8 байт)
    p->x = 100;  // Меняем оригинал через ->
}
// Быстро, можно менять
```

#### **3. По константному указателю (если не менять)**
```c
void func(const struct Point* p) {  // Только чтение
    printf("%d", p->x);
    // p->x = 100; // Ошибка! const запрещает
}
```

---

### **Пример из жизни:**

**Без структуры:**
```c
// Запутано!
save_student("Иван", 20, "ИВБО-01", 4.5, "ул. Ленина");
```

**Со структурой:**
```c
struct Student {
    char name[50];
    int age;
    char group[20];
    float gpa;
    char address[100];
};

struct Student s = {"Иван", 20, "ИВБО-01", 4.5, "ул. Ленина"};
save_student(&s);  // Чисто и понятно
```

---

### **Главное правило:**

**Маленькая структура** (2-3 поля) → можно передавать по значению  
**Большая структура** → всегда передавать по указателю

```c
// Маленькая (8 байт) - можно копировать
struct Vec2 { float x, y; };
void use_vec2(struct Vec2 v);  // OK

// Большая (100+ байт) - указатель!
struct Student { char name[100]; ... };
void use_student(struct Student* s);  // Так лучше
```

---

### **Итог в трёх пунктах:**

1. **Структура** = упаковка переменных в одну коробку
2. **Передаём** обычно по указателю (`&struct`)
3. **Не меняем**? Используем `const` перед указателем

**Проще:** Структура — как чемодан. Не тащите весь чемодан (копию), покажите квитанцию на хранение (указатель) и работайте с оригиналом.



**Структуры данных — это "упаковка" связанных данных в один блок.**

**Структура** — это способ объединить несколько переменных разного типа под одним именем.


### **Основные тезисы:**

1. **Структуры** группируют связанные данные разных типов
2. **Актуальны** для: упрощения кода, возврата нескольких значений, работы с железом
3. **Передавать лучше по указателю** (быстрее, экономит память)
4. **Используйте const** когда функция только читает данные
5. **Размер структуры** может быть больше суммы полей из-за выравнивания

**Золотое правило:** Если структура больше 8-16 байт — передавайте по указателю. Для маленьких структур (2-3 поля) можно передавать по значению.
