## Принципы функционирования системного программного обеспечения (на основе анализа отличий от других типов ПО)

**Системное ПО — это "диспетчер компьютера".**

---

### **Главная задача:**
Управлять **ресурсами компьютера** (процессор, память, диски) чтобы **программы не мешали друг другу**.

---


**Принципы функционирования системного программного обеспечения**

---

### **Классификация ПО (по уровням)**

1. **Системное ПО** — "фундамент" (ОС, драйверы, утилиты)
2. **Инструментальное ПО** — "инструменты строительства" (компиляторы, IDE)
3. **Прикладное ПО** — "здания" (браузеры, офисные пакеты)

---

### **Ключевые отличия системного ПО от прикладного**

| **Критерий** | **Системное ПО** | **Прикладное ПО** |
|--------------|------------------|-------------------|
| **Цель** | Управление ресурсами компьютера | Решение задач пользователя |
| **Пользователь** | Программист, администратор | Конечный пользователь |
| **Уровень привилегий** | Режим ядра (kernel mode) | Режим пользователя (user mode) |
| **Доступ к железу** | Прямой | Только через системные вызовы |
| **Сбой** | Критичен (падение системы) | Локальная ошибка |
| **Периодичность использования** | Постоянно | По необходимости |
| **Примеры** | Windows NT, Linux kernel, драйвер Nvidia | Word, Chrome, Photoshop |

---

### **4 главных принципа:**

1. **"Всем всё"** (Виртуализация)
   - Каждой программе кажется, что у неё **весь компьютер**
   - На самом деле ОС быстро переключает между программами

2. **"Не лезь в чужое"** (Изоляция)
   - Программы работают в **своих песочницах**
   - Одна упала — другие работают

3. **"Говори через переводчика"** (Абстракция)
   - Программа говорит: "Сохрани файл"
   - ОС переводит: "Запиши в сектор 5, дорожку 3..."
   - Программа **не знает** про железо

4. **"Очередь и приоритеты"** (Мультиплексирование)
   - Всем дают **по очереди** поработать
   - Важные задачи — вперёд

---

### **Чем отличается от других программ?**

| **Критерий** | **Системное ПО** | **Обычная программа** |
|--------------|------------------|----------------------|
| **Кто пользуется** | Другие программы | Люди |
| **Что делает** | Даёт ресурсы | Решает задачи |
| **Если сломается** | Падает весь компьютер | Падает только она |
| **Доступ к железу** | Прямой | Через ОС |

**Примеры системного ПО:**
- Windows, Linux (ОС)
- Драйвер видеокарты
- Антивирус (системная утилита)

**Примеры НЕ системного ПО:**
- Word, Chrome, Photoshop

---

### **Как работает на примере:**
```c
// Программа хочет:
printf("Привет");  // 1. Сказать "Привет"

// ОС делает:
1. Проверить, куда выводить (консоль? файл?)
2. Найти свободное место на экране
3. Подготовить символы
4. Отправить видеокарте
5. Видеокарта рисует буквы
```

**Без системного ПО:** Программы дрались бы за память, стирали файлы друг друга, компьютер зависал бы постоянно.

---

### **Ключевые тезисы**

1. **Системное ПО — посредник** между железом и прикладными программами
2. **Основная функция** — управление ресурсами (CPU, память, I/O)
3. **Ключевые принципы:** абстракция, виртуализация, мультиплексирование, изоляция
4. **Архитектуры:** монолитная (Linux), микроядерная (QNX), гибридная (Windows)
5. **Отличия от прикладного ПО:** привилегии, доступ к железу, критичность сбоев
6. **Тенденции:** контейнеризация, unikernel, безопасность через изоляцию

### **Итог одной фразой:**
**Системное ПО — это правила дорожного движения для программ, чтобы они не сталкивались и не мешали друг другу.**
